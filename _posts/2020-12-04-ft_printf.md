# 프로젝트 소개

ft_printf 프로젝트는 말그대로 우리가 지금까지 자주 써왔던, libc의 함수 `printf`를 직접 만드는 프로젝트이다.  
이를 통해 `가변인자(variadic arguments)`, `가변함수(variadic functions)`를 쓰는 방법을 주로 배우게 될 것이다.  
성공적인 ft_printf를 만드려면 체계적이고 확장성이 좋은 코드를 짜는 것이 중요하다.

# Mandatory part

| 프로그램 이름       | libftprintf.a                                                                       |
| ------------------- | ----------------------------------------------------------------------------------- |
| 제출할 파일         | \*.c, \*/\*.c, \*.h, \*/\*.h, Makefile                                              |
| Makefile            | all, clean, fclean, re, bonus                                                       |
| 외부 함수           | malloc, free, write, va_start, va_arg, va_copy, va_end                              |
| Libft 사용가능 여부 | yes                                                                                 |
| 설명                | Write a library that contains ft_printf, a function that will mimic the real printf |

- ft_printf의 프로토타입은 아래와 같아야 한다.

```c
int ft_printf(const char *, ...);
```

- libc의 printf함수를 다시 짜는 것이다.
- 실제 printf처럼, 버퍼를 핸들링(buffer management)하면 안된다.
- 다음 변환(conversions)을 핸들링한다: cspdiuxX%
- 다음 플래그(flags)의 모든 조합을 핸들링한다: ’-0.\*'와 모든 변환들의 최소 필드 너비(minimum field width)
- 실제 printf와 비교하여 평가된다.
- 라이브러리를 만들 때 `ar` 명령어를 사용해야 한다. `libtool` 명령어는 사용이 금지된다.

> man 3 printf / man 3 stdarg

# Bonus part

- Mandatory part가 완벽하지 않다면 보너스 파트에 대해 생각조차 하지 말아야한다.
- 모든 보너스를 다 할 필요는 없다.
- 변환 'nfge' 중 하나 이상을 핸들링해준다.
- 플래그 'l ll h hh' 중 하나 이상을 핸들링해준다.
- 다음의 플래그 '# +'를 모두 핸들링해준다. (가운데는 공백)
